# 🚀 Smart Inventory Management System (SIMS) - Manual Run Guide

This document provides a step-by-step guide to running the Smart Inventory Management System (SIMS) on your local machine.

## 📋 Prerequisites
Before starting, ensure you have the following installed on your system:
- **Node.js** (LTS version recommended)
- **MongoDB Server** (Running locally on the default port `27017`)

---

## ⚙️ Setup Instructions

### Phase 1: Database Initialization (First Time Only)
1. Open your terminal in the project root directory.
2. Run the seeding script to create the initial categories, products, and user accounts:
   ```bash
   node manual_seed.js
   ```
   *Note: This will clear any existing data in the `inventory_db` database and create fresh data with properly hashed passwords.*

### Phase 2: Start the Backend Server
1. In your terminal, run:
   ```bash
   npm run server
   ```
   *The server will start at `http://localhost:5000` with API endpoints accessible at `/api`.*
   *Keep this terminal window open while using the application.*

### Phase 3: Start the Frontend Application
1. Open a **new** terminal window (do not close the backend terminal).
2. Run the development server:
   ```bash
   npm run start
   ```
3. Once the command completes, access the application at:
   **[http://localhost:3000](http://localhost:3000)**

## 🛡️ Role-Based Access Control (RBAC) & Security Policies

The system enforces a strict security hierarchy:

### ⚡ Access Levels
| Role | Access Rights |
| :--- | :--- |
| **Admin** | **Master Control**: Full system access + User Management (Identity Provisioning). |
| **Manager** | **Management**: Product management, inventory reports, and supplier handling. |
| **Staff** | **Execution**: Focused purely on "Stock In" and "Stock Out" transactions. |

### 🔒 Operational Governance Rules
- **One Master Admin**: The system is restricted to **one Admin account** only. This account **cannot be deleted or modified** from the interface.
- **Admin-Led Provisioning**: Manager and Staff accounts can be created **only by the Admin**.
- **Role Elevation Block**: Regular users cannot be elevated to the Admin role.
- **Audit Logging**: Every action by every user is recorded in the system's ledger.

## 🛠️ Troubleshooting

- **Database Connection Error**: Ensure the MongoDB service is running on your machine.
- **Port Conflict**: If port 3000 or 5000 is already in use by another application, you may need to close that application or change the ports in `vite.config.ts` and `server.js`.
- **Login Failure**: If you cannot log in, run the `node manual_seed.js` command again to reset the user credentials.
